# Generation Rules

> How to generate the 4 live-generated fields using ONLY this session's user data. No templates pre-filled with generic content. No patterns from other sessions. Every generated field must be traceable to specific user answers.

---

## The 4 Generated Fields

1. **Three Directions (3)** — Screen 9, Phase B (referred to internally as "vectors"; user-facing language uses "directions/paths/options")
2. **Flow-Sized First Move** — Screen 10, Phase C
3. **Negative Surprise** — Screen 12, Phase D
4. **Your Name + 3 Gap Questions** — Screen 13, Phase D

---

## 1. Three Directions (3) — Screen 9

### Input data used
- Screen 3 answer (envy — broad list + the one-they-narrowed-to)
- Screen 2 answer (peak-moment scene — single moment, not a list)
- Screen 4 answer (pattern in plain words + character name + what it does + when it loses grip)
- Screen 5 answer (Sunday-night gut behavior)
- **Screens 2 + 4 + 5 ALSO used as a "what they're already doing" filter** — see filter rule below

### Output format

Each option is:
```
**Option [N] — [Plain-action title, no metaphor] ([cadence — daily habit / once this week / weekly]):** [1-2 sentence direction + concrete first move]
```

**Format requirements:**
- Title MUST be plain action, NOT metaphorical (e.g. "Eat one meal without judging yourself" — not "Eat One Meal On Your Own Side")
- Cadence tag REQUIRED in the title parens — daily habit / once this week / weekly
- "Vector" never appears in user-facing output

### How to derive each option

**Option 1 — derived from envy (treat envy as DIRECTION SIGNAL, not action signal).** Envy data is a clue about what quality the user wishes they had — translate the direction into a repeatable practice that teaches the underlying freedom, not a one-shot mimicry of the envied person's surface behavior.

- Ask: *What specifically did they envy?* (quality, behavior, visibility, freedom, craft, etc.)
- Ask: *What is the underlying freedom or quality that creates that?*
- Ask: *What daily / weekly practice teaches that quality in THEIR life?*
- Name the direction with a plain-action title.

**Option 2 — derived from peak-moment.** Take the conditions of their peak-moment and propose scaling them up. If they came alive at [context], the option is "more of [context], intentionally scheduled."

- Ask: *What were the conditions? (Who was there? What were they doing? What kind of problem? What kind of feedback?)*
- Ask: *What's a deliberate move to get more of those conditions this week?*
- Name the direction with a plain-action title.

**Option 3 — derived from leak ("when it loses grip").** Take the condition under which the character loses power and propose doing more of that.

- Ask: *When does the character lose grip, according to them?*
- Ask: *What's a trigger or rule that creates more of that condition?*
- Name the direction with a plain-action title.

### Generation rule — FILTER overlapping behaviors

**Before delivering, check each option against Screens 2 + 4 + 5 data.** If an option overlaps with what the user is already doing, **regenerate it.** Options must STRETCH, not CONFIRM. An option the user is already doing most days becomes the "easy pick" under fatigue and produces session output of "keep doing what you're doing" — no forward motion.

Specifically:
- Does any element of the proposed option appear in Screen 2's peak-moment scene? → too close, regenerate
- Does any element appear in Screen 4's "less hold" condition? → too close (they already access it), regenerate
- Does any element appear in Screen 5's Sunday-night default? → confirms current behavior, regenerate

### Generation rule — PREFER daily-habit options when data supports both

When the user's data could support either a daily habit or a one-shot, **default to the daily habit.** Habits compound; one-shots don't. Envy data skews toward one-shots by construction (the envied person did THE THING ONCE-VISIBLE) — Option 1 most often violates this rule and should be the most carefully reviewed against the envy-as-direction-signal rule above.

Cadence determination per option:
- **Daily habit** — practice repeats every (or most) days, building a rhythm. Default when data supports it.
- **Once this week** — single concrete action with a date. Use only when the option is genuinely a one-time gateway.
- **Weekly** — practice repeats once per week. Use when daily would be too dense for the user's stated capacity.

### Quality check before delivering the 3 options

- [ ] Each option uses language from the user's own answers (not generic coach-speak)
- [ ] Each option has a concrete first move, not abstract advice
- [ ] Each option is walkable this week, not a multi-month project
- [ ] Option 1 is clearly anchored to envy data, Option 2 to peak-moment, Option 3 to leak
- [ ] No option is "figure out what you want" or similarly vague
- [ ] **(V2)** No option overlaps with Screen 2 / 4 / 5 existing-behavior data
- [ ] **(V2)** Each option title is a plain action (no metaphors)
- [ ] **(V2)** Each option has its cadence tagged in the title parens

### What to AVOID

- Pre-filled generic options like "Creative expression" or "Deep work"
- Options that require resources the user hasn't mentioned
- Options that sound like a coaching book chapter
- Options that are fundamentally the same thing worded differently
- **(V2) Metaphor titles** — keep concept, drop poetic naming. "Eat one meal without judging yourself" beats "Eat One Meal On Your Own Side."
- **(V2) The word "vector"** — internal only, never user-facing

## 2. Flow-Sized First Move — Screen 10

### Input data used
- User's vector selection from Screen 9
- User's Screen 4 leak data (to understand the obstacle shape)
- User's Screen 1 domain selection (to ground the move in an area they care about)

### Output format

```
[Action in 1 sentence — must be 15-30 min, shippable this week, slightly above current skill]
```

### Rules

- **15-30 minutes** — not 5 min (trivial), not 2 hrs (too big to repeat)
- **Shippable this week** — a real thing produced or done, not a reflection or plan
- **Slightly above current skill** — requires a small stretch but not heroic effort
- **Specific** — "post one thing publicly" beats "be more visible"
- **Observable** — you can tell from the outside whether it happened

### Quality check

- [ ] Can the user tell me Sunday night whether they did it or not?
- [ ] Does it require less than 30 min of focused time?
- [ ] Does it produce a concrete artifact, action, or outcome?
- [ ] Is it something they could NOT have done last week without the vector?

## 3. Negative Surprise — Screen 12

> **Why this screen has its own rules:** Earlier designs defaulted to abstract-mechanism language — "indistinguishable from X from the inside," "X's shutdown move." Pilot data showed the abstract-mechanism default produced surprises the runner couldn't deliver in plain words. The version that landed reliably used concrete user-examples + short sentences + plain contrast — pattern: `[Two concrete user-examples]. [Contrast: "one X, the other Y"]. [Punchline about timing — "you only know after"]`. The current rules make that landing pattern the DEFAULT, not the fallback.

### Input data used
- Screen 7 signature (what they wrote about coming alive — full text)
- Screen 8 leak (character + behavior + when weakens — full text)
- Screen 2 peak-moment scene (for concrete user-examples)
- Screen 4a plain-words pattern description (the user's own non-metaphorical language)

### Default Template — use FIRST, not as fallback

```
[Concrete user-example A from Screen 2 or peak-moment data]. [Concrete user-example B from Screen 4 or leak data].

One [verb that names what A does — fills you, lights you, etc.]. The other just [verb that names what B does — numbs, drains, hides, etc.].

[Punchline about timing — typically "you only know after," "you can't tell while you're in it," "the difference shows up the next day," etc.]
```

### Absolute output constraints

**(A) Concrete nouns only.** Banned abstract nouns: *mechanism, move, dynamic, pattern (when used as the noun being described), shadow, shutdown.* Use the user's actual nouns from their session — actual things they wrote about (a meal, a Zoom call, a workout, a scroll session, etc.).

**(B) Short sentences.** Average sentence length ≤12 words. If you write a sentence over 15 words, break it.

**(C) User's own examples REQUIRED.** Pull at least 2 specific examples from THIS session's user data — not from a template, not invented. Names of activities they mentioned. Places they mentioned. People they mentioned. If you can't find 2 concrete user-examples, you don't have enough signal — ask one clarifying question instead of generating.

**(D) Contrast structure.** Two examples → "one X, the other Y" naming the divergence → punchline about TIMING (the "you only know after" beat). This is the landing shape.

**(E) Max length: ~4 sentences.**

### Forbidden patterns

These specific phrasings are banned from V2:

- ❌ "Indistinguishable from X"
- ❌ "From the inside"
- ❌ "X's shutdown move"
- ❌ "Real presence and pure escape"
- ❌ Any clinical / abstract / therapist-language framing
- ❌ "Your strength under stress" or "Your biggest strength" as opener (too generic)

### Quality gate — RUNNER asks itself BEFORE delivery

> *"If I read this to someone who didn't do the session, would they understand it without context?"*

- If **NO** → rewrite. Pilot data showed the runner couldn't explain abstract-mechanism drafts; this rule prevents that failure mode at generation time.
- If **YES** → deliver.

The runner must be able to explain the surprise in plain words to an outsider. If you couldn't, the surprise isn't ready.

### Example — failed draft vs. landing template

**Failed (abstract-mechanism — too hard to parse, runner couldn't explain it cleanly):**
> "Your biggest strength — engaging deeply when others bring energy — is indistinguishable from The Hamster in a Wheel from the inside. Real presence and pure escape feel identical while you're in the state."

**Landed (concrete-noun contrast — same user, third pass):**
> "Real rest and hiding from life feel the same in the moment. But one fills you up. The other just numbs you."

Note how the second version lands: concrete contrast (rest vs hiding) + plain verb (fills vs numbs) + timing punchline (the moment / how they differ shows up later) — no character name needed, no abstract-mechanism vocabulary.

### Quality check before delivery

- [ ] Two concrete user-examples present (from Screens 2 or 4)
- [ ] No banned abstract nouns
- [ ] All sentences ≤15 words
- [ ] Contrast structure present (one X, the other Y)
- [ ] Punchline names timing
- [ ] Runner can explain in plain words to an outsider
- [ ] No softening ("to be fair...", "but also...")
- [ ] No advice / no resolution / no fix
- [ ] Max ~4 sentences

### What to AVOID

- Generic shadow-work framings ("your strengths are also your weaknesses")
- Templates that could apply to anyone
- Softening language ("to be fair...", "this is common...")
- Advice or fix suggestions — the negative surprise stands alone, no resolution
- Abstract-mechanism framings (see Forbidden Patterns above)
- Using the character name unnecessarily — sometimes plain contrast lands without invoking the character (the landing example above omitted the character name)

## 4. Your Name + 3 Gap Questions — Screen 13

### A. Your Name (the 3-5 word handle)

#### Input data used
- Screen 7 signature (strength)
- Screen 8 leak (what's in the way)
- Screen 6 chapter title (their own framing of their current life)

#### Output format

```
**[Qualifier] [Identity Noun]**

e.g. "The Quiet Builder," "The Rehearsing Starter," "The Hidden Operator"
```

#### Construction rules

1. **It's a HANDLE, not a verdict.** The user confirms it lands or writes reasons it feels off (no rename inside this session). Do not write it as if final. The reasons-it-feels-off become Session 2 input instead. See Screen 13 in `session-1-script.md` for the 13a flow.
2. **3-5 words max.** Short enough to hold in one breath.
3. **Captures the tension, not just the strength.** A good Your Name reveals the interplay between their signature AND leak, not one or the other.
4. **Uses a qualifier that's specific, not generic.** "The Providing Stranger" beats "The Provider." "The Rehearsing Starter" beats "The Beginner."
5. **Barnum-calibrated.** Sticky enough that they'll remember it. Specific enough that it couldn't apply to 80% of people.
6. **(Generation implication of the no-rename rule)** Calibrate conservatively — since the user can't reshape the name on the spot, the runner-generated version needs to be solid enough to land for ~70%+ of users on first read OR clearly invite reasons-it-feels-off (which is also valuable Session 2 data). Avoid hyper-clever names that nail it for 30% and feel weird to 70%.

#### Construction method

- Ask: *What's the 1-word identity that fits their signature?* (Builder, Explorer, Connector, Operator, Creator, Seeker, etc.)
- Ask: *What's the 1-word qualifier from their leak or chapter title?* (Undeclared, Hidden, Rehearsing, Paused, Quiet, Half-Formed, etc.)
- Combine. Test by reading it back: does it capture both the strength and the tension?

#### Quality check

- [ ] 3-5 words
- [ ] Couldn't apply to 80% of random people
- [ ] Captures both signature AND leak
- [ ] Uses an uncommon qualifier, not "The Creative"
- [ ] User could reject and rename (and probably will iterate)

### B. 3 Gap Questions

#### Purpose
Generate 3 open questions that Session 2 will need to answer. These compound the hook ("we don't know yet if X, so come back") AND give Session 2 concrete material to open with.

#### Input data used
- ANY Phase A answer where the user was thin, surprising, or ambiguous
- Anything they said that opens a door they didn't walk through

#### Output format

```
- [Open question, 1 sentence, specific to their data]
```

#### Construction rules

1. **Must be open-ended.** Not yes/no. Not "do you X or Y?"
2. **Must be specific to THIS user's session.** No generic "what are your core values?" questions.
3. **Must reference their data.** Use their character name, their peak-moment context, their chapter title.
4. **Must be answerable in Session 2** — not so broad that no session could handle it.
5. **Lean toward cross-domain extension.** If Phase A data is heavy in work, ask about relationships. If it's heavy in one life area, check the others.

#### Types of good gap questions

- **Cross-domain:** "Does [their named leak character] show up in [domains they didn't explore]?"
- **Counter-part:** "You named [X] as the amplifier. Who drains you?"
- **Specificity probe:** "Your envy pointed at [X] — is that really about [visible thing], or about [underlying thing]? Those are different."
- **Threshold question:** "You said [pattern has less grip when Y]. What's the smallest version of Y you could schedule this week?"
- **Chapter-title follow-through:** "Your chapter title is [X]. What has to be true at the end of this chapter for it to have been worth the title?"

#### Quality check

- [ ] 3 questions total
- [ ] Each question references user-specific data (names, characters, contexts)
- [ ] Each is open, not yes/no
- [ ] Session 2 could plausibly open with any of them
- [ ] Not generic ("what are your values?" is banned)

## General Rules for All Generation

### Before you generate, check:
- Do I have the user's verbatim answers in the session file?
- Am I using their vocabulary, not mine?
- Is there at least one specific word or phrase from their answer in my output?

### After you generate, check:
- Could this have been generated without this user's data? (If yes, it's too generic.)
- Would this make sense to them specifically, or to anyone? (Answer: only them.)
- Did I soften anything? (If yes, restore.)

### If you don't have enough signal to generate

Ask ONE clarifying question. Format: *"Quick one before I pull this together — [specific question tied to their earlier answer]?"*

Do not:
- Make up content to fill the gap
- Use a generic placeholder
- Skip the field

## Calibration examples — good vs. bad

### Three Directions (Screen 9) — calibration

**BAD V1-style (generic title, no cadence, no filter against existing behavior):**
> Option 1 — Creative Output: Share your work more publicly this week.

**BAD (metaphor title, no cadence tag):**
> Option 1 — Public Shipping: Post one markdown note or screenshot of work-in-progress to a place at least one person you respect will see it.
>
> *(Why banned in V2: "Public Shipping" is a metaphor; user has to decode what it means before deciding. No cadence tag.)*

**GOOD (plain-action title + cadence + filtered against existing behavior):**
> Option 1 — Post one work-in-progress screenshot publicly (once this week — one post): Pick one piece of work you've already started — a markdown note, a screenshot, a half-built thing — and put it somewhere at least one person you respect will see it. Doesn't matter what; just that it has your name on it.
>
> *(Why this lands: title describes the action directly — "Post one work-in-progress screenshot publicly" — no decoding needed. Cadence tagged: "once this week — one post." First-move is concrete. The user can't already be doing this if Screen 5 didn't surface "I share my work publicly" as their Sunday-night default.)*

---

### Negative Surprise

**BAD (generic — banned):**
> Your strengths can also become your weaknesses if overused.

**BAD (abstract-mechanism — banned):**
> Your biggest strength — going deep alone on a problem you care about — is indistinguishable from The Quiet Hours character from the inside. Deep-solo building and deep-solo scrolling feel identical while you're in the state. You can't tell them apart until after.
>
> *(Why banned: "indistinguishable from X," "from the inside," abstract-mechanism vocabulary the runner can't deliver in plain words.)*

**GOOD (concrete contrast + plain verbs + timing punchline):**
> Building something hard and scrolling for an hour can feel the same in the moment. One fills you up. The other just numbs you. You only know which it was the next morning.
>
> *(Why this lands: concrete user-examples — "building something hard" + "scrolling for an hour" — short sentences, plain contrast, timing punchline. No character name needed; no abstract-mechanism vocabulary. This is the default template, not a fallback.)*

### Your Name (handle)

**BAD (generic):**
> The Creative

**GOOD (specific):**
> The Providing Stranger

(The qualifier "Providing" captures their strength — they give consistently to crew and family — while "Stranger" captures the leak: present for everyone but never landing in their own skin. 3 words, couldn't apply to most people.)

### Gap Questions

**BAD (generic):**
> - What are your core values?

**GOOD (specific):**
> - Does El Viejo only show up at home, or does he also run your work decisions when you're tired?
> - You named your foreman as the one person who pushes back when you're wrong. Who else pushes back — and who never does?
> - Your envy pointed at men who chose family time over a bigger paycheck — is that really about the time itself, or about being PRESENT when you're there? Those are different.
